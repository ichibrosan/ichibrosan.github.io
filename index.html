Introduction to the Virtual Protocol Adapter and FrameWork

Description

The Virtual Protocol Adapter and Multiware Framework is a long-term project undertaken by Douglas Goodall.

Virtual Protocol Adapter

The Virtual Protocol Adapter is a software component that acts as a bridge between disparate hardware and software. It is designed to provide a straightforward set of interfaces that may be used by to communicate with other hardware and software components.

As the Virtual Protocol Adapter is an application program written in C++ and designed to run on the Linux operating system, the VPA and it's utilities may call any API's normally available to a Linux application as well as those supplied by additional libraries, dynamic, or static. In addition to the regular C library, the VPA accesses several commonly used libraries, those being pthread, and cgicc and uuid. The VPA also makes heavy use of the Goodall Multiware Framework, a library comprised of many classes that provide additional functionality that the VPA required to fulfill its design.

For example, the VPA has access to the System V Shared Memory services that are supplied in Linux. This shared memory region contains a variety of variables and data structures that provide for the control and status of the VPA and it's associated components.

It is the "CSysLog" that provides the member functions to write messages into the system logs.

It is the "shared" C++ class that provides the member functions that may be used to establish addressability to the shared region. The structure definition for this shared memory region is called shmemng.h. This structure contains a variety of variables that aid in the construction of well-formed function request packets used with the Remote Procedure Call feature.
